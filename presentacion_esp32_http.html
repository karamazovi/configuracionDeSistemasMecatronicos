Paso 1 · Librerías


#include <WiFi.h>          // Conexión a red WiFi
#include <WiFiClient.h>    // Cliente TCP (usado internamente)
#include <WebServer.h>     // Servidor HTTP del ESP32
#include <uri/UriBraces.h> // Rutas dinámicas /ruta/{param}

WiFi.h
Gestiona la conexión WiFi — escanea redes, conecta, obtiene IP.

WebServer.h
Servidor HTTP completo integrado en Arduino ESP32. Registra rutas y responde peticiones.

UriBraces.h
Extiende WebServer para soportar rutas con parámetros variables como /toggle/{n}.

WiFiClient.h
Clase base TCP. WebServer la usa internamente para gestionar cada conexión entrante.

Paso 2 · Configuración global


// ── Credenciales WiFi ─────────────────────────────────
#define WIFI_SSID     "Wokwi-GUEST"
#define WIFI_PASSWORD ""
#define WIFI_CHANNEL  6   // Fijar canal acelera la conexión

// ── Servidor en puerto 80 (HTTP estándar) ─────────────
WebServer servidor(80);

// ── Pines físicos de los LEDs ─────────────────────────
const int LED1 = 26;
const int LED2 = 27;

// ── Estado en memoria (false = apagado) ───────────────
bool estadoLed1 = false;
bool estadoLed2 = false;

#define crea constantes en tiempo de compilación — sin costo en RAM.

WebServer servidor(80) instancia el servidor. El 80 es el puerto HTTP estándar.

Las variables bool guardan si cada LED está encendido. El pin físico se actualiza cuando cambian.

Paso 3 · Función sendHtml() — HTML embebido


void sendHtml() {
  String respuesta = R"(
    <!DOCTYPE html><html>
      <head>
        <title>Demo Servidor Web ESP32</title>
        <style>
          .btn { background-color: #5B5; color: #fff; font-size: 2em; }
          .btn.OFF { background-color: #333; }
        </style>
      </head>
      <body>
        <h1>ESP32 Web Server</h1>
        <a href="/toggle/1" class="btn TEXTO_LED1">TEXTO_LED1</a>
        <a href="/toggle/2" class="btn TEXTO_LED2">TEXTO_LED2</a>
      </body>
    </html>
  )";

R"(...)" = raw string literal.
Permite escribir HTML con comillas y saltos de línea sin escapar nada.

TEXTO_LED1 y TEXTO_LED2 son marcadores de posición que se reemplazan dinámicamente antes de enviar.

Paso 3 · Función sendHtml() — replace y envío


  // Reemplazar marcadores con el estado real
  respuesta.replace("TEXTO_LED1", estadoLed1 ? "ON" : "OFF");
  respuesta.replace("TEXTO_LED2", estadoLed2 ? "ON" : "OFF");

  // Responder al navegador con HTTP 200 + contenido HTML
  servidor.send(200, "text/html", respuesta);
}

respuesta.replace(A, B) reemplaza todas las ocurrencias de A por B dentro del String.
El marcador aparece dos veces en el HTML (en class y en texto), ambas se actualizan.

servidor.send(200, "text/html", respuesta)
200 = código HTTP "OK"
"text/html" = tipo MIME
respuesta = cuerpo del mensaje

Operador ternario: estadoLed1 ? "ON" : "OFF"
Si estadoLed1 es true → produce "ON", si es false → produce "OFF".

Paso 4 · setup() — Serial y GPIO


void setup(void) {
  Serial.begin(115200);   // Puerto serie a 115200 bps para depuración
  pinMode(LED1, OUTPUT);  // GPIO 26 → modo salida
  pinMode(LED2, OUTPUT);  // GPIO 27 → modo salida

Serial.begin(115200)
Abre el puerto serie. El 115200 es la velocidad en baudios (bits por segundo).

pinMode(pin, OUTPUT)
Configura el GPIO como salida digital. Sin esto, el pin está en alta impedancia.

void setup(void)
Arduino llama esta función una sola vez al arrancar. Aquí va toda la inicialización.

Paso 5 · setup() — Conexión WiFi


  WiFi.begin(WIFI_SSID, WIFI_PASSWORD, WIFI_CHANNEL);
  Serial.print("Conectando a WiFi ");
  Serial.print(WIFI_SSID);

  // Esperar hasta obtener conexión (bucle bloqueante)
  while (WiFi.status() != WL_CONNECTED) {
    delay(100);
    Serial.print(".");
  }
  Serial.println(" ¡Conectado!");
  Serial.print("IP: ");
  Serial.println(WiFi.localIP());

WiFi.begin(ssid, pass, canal)
Inicia el proceso de asociación con el punto de acceso. Proporcionar el canal evita el escaneo completo → conexión más rápida.

while (WiFi.status() != WL_CONNECTED)
Bucle de espera activa. Revisa el estado cada 100 ms e imprime un punto. Cuando la IP es asignada, WL_CONNECTED se cumple y el bucle termina.

WiFi.localIP() devuelve la dirección IP asignada por el router (o simulador Wokwi). Desde esa IP se accede al servidor.

Paso 6 · setup() — Registrar rutas HTTP


  // Ruta raíz "/" → muestra la página principal
  servidor.on("/", sendHtml);

  // Ruta dinámica con parámetro → cambia estado del LED
  servidor.on(UriBraces("/toggle/{}"), []() {
    String led = servidor.pathArg(0);   // Captura el número entre llaves
    Serial.print("Cambiar LED #");
    Serial.println(led);

    switch (led.toInt()) {
      case 1:
        estadoLed1 = !estadoLed1;
        digitalWrite(LED1, estadoLed1);
        break;
      case 2:
        estadoLed2 = !estadoLed2;
        digitalWrite(LED2, estadoLed2);
        break;
    }
    sendHtml();  // Responder con la página actualizada
  });

  servidor.begin();

servidor.on(ruta, handler)
Asocia una URL con una función que se ejecuta al recibir esa petición.

UriBraces("/toggle/{}")
Las llaves {} capturan cualquier texto. pathArg(0) lo recupera.

!estadoLedX = toggle.
digitalWrite aplica ese nuevo estado al pin físico (HIGH/LOW).

Paso 7 · loop() — El corazón del servidor


void loop(void) {
  servidor.handleClient();  // Revisar y procesar peticiones HTTP
  delay(2);                 // 2 ms de pausa para estabilidad
}

servidor.handleClient()
Verifica si algún cliente (navegador) envió una petición HTTP.
Si hay una → la enruta al handler registrado con servidor.on().
Debe llamarse continuamente para que el servidor responda.

delay(2)
Pausa de 2 milisegundos.
Sin ella el loop correría sin descanso y podría desestabilizar el RTOS del ESP32.
No es perceptible para el usuario.

⚙️ loop() en Arduino se ejecuta millones de veces por segundo. Todo lo que pase aquí debe ser rápido y no bloqueante.

Flujo completo de una petición

1

El usuario abre http://localhost:8180 → el navegador envía GET /

2

handleClient() detecta la petición y llama a sendHtml()

3

sendHtml() construye el HTML, reemplaza ON/OFF según estado, responde con HTTP 200

4

El usuario hace clic en un botón → navegador envía GET /toggle/1

5

Handler captura pathArg(0) = "1", invierte estadoLed1, llama digitalWrite()

6

sendHtml() vuelve a enviar la página con el estado actualizado → el botón cambia de color

Conceptos clave

Raw String Literal R"(...)"

Permite escribir texto multilínea con comillas sin escapar. Ideal para HTML embebido en C++.

Operador Toggle !variable

Invierte un booleano: true → false → true. Patrón estándar para encender/apagar.

Rutas dinámicas UriBraces

Captura parámetros en la URL. Un solo handler atiende /toggle/1 y /toggle/2.

Lambda []() { ... }

Función anónima definida en el lugar. Útil cuando el handler es corto y no se reutiliza.

Patrón Request → Toggle → Respond

El ESP32 actúa simultáneamente como servidor web y como controlador de hardware. Sin MQTT, sin backend externo.
