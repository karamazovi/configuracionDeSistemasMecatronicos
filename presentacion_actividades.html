A1 · Paso 1 — Librerías


#include <WiFi.h>          // Conexión a red WiFi
#include <WiFiClient.h>    // Cliente TCP (interno a WebServer)
#include <WebServer.h>     // Servidor HTTP integrado del ESP32
#include <uri/UriBraces.h> // Rutas con parámetros: /ruta/{param}

WiFi.h — gestiona conexión: escanea, asocia, obtiene IP.

WebServer.h — servidor HTTP completo. Registra rutas y despacha peticiones.

UriBraces — extiende WebServer para soportar /ruta/{n}. Captura el valor con pathArg(0).

WiFiClient.h — clase TCP base que WebServer usa internamente por cada conexión.

A1 · Paso 2 — Configuración global


#define WIFI_SSID     "Wokwi-GUEST"
#define WIFI_PASSWORD ""
#define WIFI_CHANNEL  6        // Canal fijo → conexión más rápida

WebServer servidor(80);        // Puerto 80 = HTTP estándar

const int LED1 = 26;           // GPIO del LED 1
const int LED2 = 27;           // GPIO del LED 2

bool estadoLed1 = false;       // false = apagado
bool estadoLed2 = false;

#define crea constantes en tiempo de compilación — cero costo en RAM.

WebServer servidor(80) instancia el servidor. El 80 es el puerto HTTP estándar.

Las variables bool reflejan el estado lógico. El pin físico se actualiza con digitalWrite.

A1 · Paso 3 — sendHtml(): HTML dinámico


void sendHtml() {
  String respuesta = R"(
    <a href="/toggle/1" class="btn LED1_TEXT">LED1_TEXT</a>
    <a href="/toggle/2" class="btn LED2_TEXT">LED2_TEXT</a>
  )";

  // Reemplazar marcadores con el estado real
  respuesta.replace("LED1_TEXT", estadoLed1 ? "ON" : "OFF");
  respuesta.replace("LED2_TEXT", estadoLed2 ? "ON" : "OFF");

  servidor.send(200, "text/html", respuesta);  // HTTP 200 OK
}

R"(...)" raw string literal — HTML con comillas sin escapar nada.

.replace(A,B) cambia todas las ocurrencias. El marcador aparece en class y en el texto.

servidor.send(200, "text/html", body) — código, tipo MIME y cuerpo.

A1 · Paso 4 — setup(): WiFi y rutas


void setup() {
  Serial.begin(115200);
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);

  WiFi.begin(WIFI_SSID, WIFI_PASSWORD, WIFI_CHANNEL);
  while (WiFi.status() != WL_CONNECTED) { delay(100); }
  Serial.println(WiFi.localIP());

  servidor.on("/", sendHtml);

  servidor.on(UriBraces("/toggle/{}"), []() {
    String led = servidor.pathArg(0);
    if (led == "1") { estadoLed1 = !estadoLed1; digitalWrite(LED1, estadoLed1); }
    if (led == "2") { estadoLed2 = !estadoLed2; digitalWrite(LED2, estadoLed2); }
    sendHtml();
  });

  servidor.begin();
}

while (WiFi.status() != WL_CONNECTED) — bucle bloqueante, espera la IP del DHCP.

Lambda [](){...} — función anónima usada como handler. !estadoLed1 = toggle en una línea.

void loop() {
  servidor.handleClient();  // Procesar peticiones HTTP entrantes
  delay(2);                 // Estabilidad del RTOS
}

A2 · Teoría — Motor a Pasos

¿Qué es?

Motor eléctrico que gira en incrementos discretos (pasos). Cada paso mueve el eje un ángulo fijo y preciso.

NEMA 17

200 pasos/vuelta → 1.8° por paso
4 bobinas (bipolar) → pines A+, A–, B+, B–

Full-step (4 fases)

Se activan las bobinas en secuencia:
AB → A̅B → A̅B̅ → AB̅ → AB...

PasoA+A–B+B–
1▲▲
2▼▲
3▼▼
4▲▼

RPM = pasos/seg ÷ 200 × 60
A 10 RPM → ~33 pasos/seg → 1 paso cada 30 ms

A2 · Librería Stepper.h


#include <Stepper.h>

// Constructor: pasos/vuelta, IN1(A+), IN2(B+), IN3(A-), IN4(B-)
Stepper motor(200, 5, 17, 18, 16);

// En setup():
motor.setSpeed(10);   // Fijar velocidad en RPM

// En un handler:
motor.step(50);       // +50 pasos = sentido horario
motor.step(-50);      // -50 pasos = sentido antihorario

Stepper(N, p1, p2, p3, p4)
N = pasos por vuelta
Orden pines: IN1, IN2, IN3, IN4

setSpeed(rpm)
Calcula el delay entre pasos. Llamar antes del primer step().

⚠ step() es bloqueante
El ESP32 no atiende peticiones HTTP mientras gira. Mantener pasosPorClick pequeño.

El orden de los pines en el constructor de Stepper.h es IN1, IN2, IN3, IN4, que corresponde a A+, B+, A–, B–. Intercambiar A– con B+ invierte el sentido de giro.

A2 · Paso 1 — Variables globales ampliadas


// ── Heredado de Actividad 1 ─────────────────────────────
WebServer servidor(80);
const int LED1 = 26, LED2 = 27;
bool estadoLed1 = false, estadoLed2 = false;

// ── Nuevo en Actividad 2 ────────────────────────────────
#define PASOS_POR_VUELTA 200
Stepper motor(PASOS_POR_VUELTA, 5, 17, 18, 16);

int velocidadRPM  = 10;   // Velocidad inicial
int pasosPorClick = 50;   // Pasos por clic de botón

Stepper motor(...) es una variable global — el objeto persiste entre peticiones HTTP y mantiene internamente el estado de la secuencia de bobinas.

pasosPorClick e velocidadRPM son variables de estado que el usuario puede modificar desde la web con rutas como /stepper/speed/15.

A2 · Paso 2 — Nuevas rutas del servidor

Ruta HTTPAcción
GET /Página principal (LEDs + Motor)
GET /toggle/{n}Invertir LED 1 ó 2 (igual que Act. 1)
GET /stepper/cwGirar motor horario pasosPorClick pasos
GET /stepper/ccwGirar motor antihorario pasosPorClick pasos
GET /stepper/speed/{n}Cambiar velocidad (1–60 RPM)
GET /stepper/steps/{n}Cambiar pasos por clic (1–400)
 

servidor.on("/stepper/cw", []() {
  motor.step(pasosPorClick);   // positivo → horario
  enviarHtml();
});

servidor.on("/stepper/ccw", []() {
  motor.step(-pasosPorClick);  // negativo → antihorario
  enviarHtml();
});

servidor.on(UriBraces("/stepper/speed/{}"), []() {
  int rpm = servidor.pathArg(0).toInt();
  if (rpm >= 1 && rpm <= 60) {
    velocidadRPM = rpm;
    motor.setSpeed(velocidadRPM);
  }
  enviarHtml();
});

A2 · Flujo completo

1

2

Clic en CW ▶ → navegador envía GET /stepper/cw

3

handleClient() despacha al handler de /stepper/cw

4

motor.step(50) — bloqueante ~150 ms — mueve el eje 50 pasos en Wokwi

5

enviarHtml() construye y envía la página con estado actualizado

6

Clic en 15 RPM → GET /stepper/speed/15 → motor.setSpeed(15) → página nueva

⚠️ Durante motor.step() el servidor no responde. Con 50 pasos a 10 RPM ≈ 150 ms de bloqueo — imperceptible para el usuario.
