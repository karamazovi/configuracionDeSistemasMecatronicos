¿Qué es un Robot Seguidor de Línea?

Definición: robot autónomo que detecta y sigue una trayectoria marcada en el suelo, sin intervención humana.
¿Cómo lo hace? Sensores IR leen el contraste blanco/negro del suelo → microcontrolador decide → motores corrigen.
Aplicaciones reales: líneas de ensamblaje, almacenes automatizados (AGVs), competencias de robótica.

¿Qué hace? Emite luz infrarroja y mide cuánta regresa. La salida analógica varía según la reflectividad del suelo.

Solo activa con ADC 0, 1, 2 o 3 → muy estricto, ideal en Tinkercad.

¿Por qué "activo bajo"?

  • La salida baja (0V) indica que detectó la línea
  • Contraintuitivo al principio pero es el estándar del sensor
  • Negro = 0 V = sensor activo = variable true

Escala ADC (10 bits)

  • 0 V → ADC = 0
  • 2.5 V → ADC = 512
  • 5 V → ADC = 1023
  • Resolución: 5 V ÷ 1024 = 4.88 mV por paso

Tabla de Verdad — Sensores de Línea

SL = A0 (izquierdo) · SR = A1 (derecho) · 1 = ve línea negra (ADC < 4)

SLSRSituación del robotAcción
00 Ambos sobre blanco — robot centrado ▶ Avanzar
01 SR cayó en negro — robot fue a la derecha ↺ Girar izquierda
10 SL cayó en negro — robot fue a la izquierda ↻ Girar derecha
11 Ambos en negro — fuera del camino ⏹ Detener
Lógica de giro: cuando SR ve negro, el robot se desvió a la derecha. Para volver → girar izquierda (hacia donde hay blanco).
Los sensores bordan la línea. Cuando están bien calibrados, normalmente ambos ven blanco (SR=SL=0) y el robot avanza recto.

Problema: loop() se ejecuta miles de veces por segundo. Sin control de tiempo, la acción cambia antes de que el motor reaccione físicamente.
Solución: delay(20) al final del loop garantiza que cada acción dure mínimo 20 ms → frecuencia de ciclo = 50 Hz.
¿Por qué 20 ms? Tiempo de respuesta mecánica del motor. Suficiente para reaccionar, sin perder agilidad en curvas.

Flujo del loop()

① Leer A0 y A1 con analogRead()

② Comparar con umbral → bool sl, sr

③ Evaluar tabla de verdad → llamar función

④ delay(20) — CPU bloqueada 20 ms

Etapa 01 — Funciones de Movimiento

Cada función escribe el estado correcto en los 4 pines del L293D.


void avanzar() {
  digitalWrite(pinIn1, HIGH); // IN1=H
  digitalWrite(pinIn2, LOW);  // IN2=L → motor izq fwd
  digitalWrite(pinIn3, HIGH); // IN3=H
  digitalWrite(pinIn4, LOW);  // IN4=L → motor der fwd
}

void girarDerecha() {
  digitalWrite(pinIn1, HIGH); // motor izq fwd
  digitalWrite(pinIn2, LOW);
  digitalWrite(pinIn3, LOW);  // motor der frenado
  digitalWrite(pinIn4, LOW);
}

void girarIzquierda() {
  digitalWrite(pinIn1, LOW);  // motor izq frenado
  digitalWrite(pinIn2, LOW);
  digitalWrite(pinIn3, HIGH); // motor der fwd
  digitalWrite(pinIn4, LOW);
}

void detener() {
  digitalWrite(pinIn1, LOW);  // todos LOW
  digitalWrite(pinIn2, LOW);  // → freno suave
  digitalWrite(pinIn3, LOW);
  digitalWrite(pinIn4, LOW);
}

avanzar()

Ambos motores hacia adelante. IN1=H·IN2=L y IN3=H·IN4=L.

girarDerecha()

Motor izquierdo avanza. Motor derecho frenado. Robot pivota sobre el eje derecho.

girarIzquierda()

Motor derecho avanza. Motor izquierdo frenado. Robot pivota sobre el eje izquierdo.

detener()

Todos LOW. El puente H aplica freno regenerativo suave.

Etapa 01 — setup() y loop()


const int pinSl  = A0;  // sensor izquierdo
const int pinSr  = A1;  // sensor derecho
const int umbral  = 4;  // ADC < 4 → línea negra
const int cicloMs = 20; // 50 Hz — duración mínima

void setup() {
  pinMode(pinIn1, OUTPUT); pinMode(pinIn2, OUTPUT);
  pinMode(pinIn3, OUTPUT); pinMode(pinIn4, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  bool sl = analogRead(pinSl) < umbral;
  bool sr = analogRead(pinSr) < umbral;

  if      (!sl && !sr) avanzar();
  else if (!sl &&  sr) girarIzquierda();
  else if ( sl && !sr) girarDerecha();
  else                 detener();

  delay(cicloMs);  // ← garantiza tiempo de acción
}

Línea 21

delay() al final. CPU bloqueada. En esta etapa es aceptable — no hay más tareas.

¿Por qué delay() se queda corto?

Escenario futuro: necesitamos medir ultrasonido (cada 20 ms) Y leer sensores IR (cada 20 ms) al mismo tiempo. Con delay(), mientras uno espera, el otro no puede ejecutarse.

Con delay() — bloqueante

  • CPU paralizada durante todo el delay
  • No puede leer sensores mientras espera
  • No puede mover el servo mientras mide
  • No puede enviar Serial mientras actúa
  • Robot "ciego" en cada pausa

Con millis() — no bloqueante

  • CPU libre siempre — loop() corre sin parar
  • Múltiples tareas con distintos periodos
  • Sensores siempre respondiendo
  • Servo y motores coordinados
  • Robot siempre "despierto"
millis() devuelve los milisegundos desde que el Arduino encendió. Crece continuamente. No detiene nada. Solo es un número.
02
Loop no bloqueante con millis()
seguidorLinea02.ino

unsigned long tAnterior   = 0;       // marca de tiempo del último ciclo
const unsigned long intervaloMs = 20;// periodo: 20 ms → 50 Hz

void loop() {
  unsigned long tAhora = millis();   // ¿cuánto tiempo ha pasado?

  if (tAhora - tAnterior >= intervaloMs) { // ¿pasaron los 20 ms?
    tAnterior = tAhora;                    // reiniciar contador

    // ── lógica de sensores ──────────────
    bool sl = analogRead(pinSl) < umbral;
    bool sr = analogRead(pinSr) < umbral;

    if      (!sl && !sr) avanzar();
    else if (!sl &&  sr) girarIzquierda();
    else if ( sl && !sr) girarDerecha();
    else                 detener();
  }
  // ← loop() sigue corriendo sin bloquear
}
tAhora - tAnterior: mide el tiempo transcurrido sin detener la CPU. Funciona aunque millis() desborde (~49 días) por aritmética modular.
Patrón universal: este mismo esquema se usa para controlar LEDs, servos, comunicación, PID — cada tarea con su propio tAnterior e intervalo.
03
+ HC-SR04 — Evasión de Obstáculos
seguidorLinea03.ino

¿Cómo funciona?

  • Emisor: emite pulso ultrasónico (40 kHz)
  • Pulso rebota en el obstáculo
  • Receptor: detecta el eco de vuelta
  • Tiempo de vuelo → distancia
Fórmula:
Distancia = (duración µs × 0.034) ÷ 2
0.034 = velocidad sonido en cm/µs · ÷2 = ida y vuelta
Timeout 25 000 µs: si no hay eco en 25 ms → devuelve 999.0 (camino libre hasta donde alcanza el sensor).

Etapa 03 — medirDistancia() y Evasión


float medirDistancia() {
  digitalWrite(pinTrigger, LOW);
  delayMicroseconds(2);           // limpiar señal
  digitalWrite(pinTrigger, HIGH);
  delayMicroseconds(10);          // pulso de 10 µs
  digitalWrite(pinTrigger, LOW);
  long d = pulseIn(pinEcho, HIGH, 25000); // timeout 25 ms
  return d == 0 ? 999.0 : d * 0.034 / 2.0;
}

void evitarObstaculo() {
  detener();
  retroceder();   delay(tRetroceso); // 700 ms atrás
  girarDerecha(); delay(tGiro);      // 500 ms giro der
  detener();
}

void loop() {
  if (tAhora - tAnterior >= intervaloMs) {
    tAnterior = tAhora;

    float dist = medirDistancia();

    if (dist < distMin) {      // distMin = 25 cm
      evitarObstaculo();
      return;                  // reiniciar ciclo
    }
    seguirLinea();             // tabla de verdad
  }
}

Servo — Señal PWM de Control

¿Cómo mueve el servo un ángulo específico?
La anchura del pulso HIGH (1–2 ms) define el ángulo. El periodo es siempre 20 ms.
0° → pulso de 1.0 ms
90° → pulso de 1.5 ms
180° → pulso de 2.0 ms

Periodo total = 20 ms (50 Hz). La librería Servo.h genera esto automáticamente.

Librería Servo.h


#include <Servo.h>
Servo servoRadar;

// En setup():
servoRadar.attach(11); // pin señal
servoRadar.write(90);  // posición inicial
delay(500);            // esperar al frente

// En loop():
servoRadar.write(0);   // girar a 0°
delay() tras write(): el servo necesita tiempo físico para alcanzar el ángulo pedido. 200–300 ms es suficiente para micros servos.

Etapa 04 — Código escanearRadar()


// Parámetros angulares del radar
const int angIzq    = 0;    // 0°   → izquierda
const int angFrente = 90;   // 90°  → frente
const int angDer    = 180;  // 180° → derecha

Servo servoRadar;

// float& → paso por referencia: modifica variables del llamador
void escanearRadar(float &distIzq, float &distDer) {
  servoRadar.write(angIzq); delay(300); distIzq = medirDistancia();
  servoRadar.write(angDer); delay(300); distDer = medirDistancia();
  servoRadar.write(angFrente); delay(200); // volver al frente
}

void evitarObstaculo() {
  detener();
  float dIzq, dDer;
  escanearRadar(dIzq, dDer);  // llenar distancias

  retroceder(); delay(tRetroceso); detener();

  if (dIzq > dDer) {
    girarIzquierda();          // más espacio a la izquierda
    Serial.println("→ izquierda");
  } else {
    girarDerecha();            // más espacio a la derecha
    Serial.println("→ derecha");
  }
  delay(tGiro); detener();
}

El robot elige el lado con mayor distancia. Funciona en espacios abiertos y laberintos simples.

05
Control PID + PWM Diferencial
seguidorLinea05.ino — ENA=9, ENB=10
Problema con Etapas 1–4: los giros son digitales (todo o nada). El robot oscila bruscamente alrededor de la línea.
Idea del PID: en lugar de girar, ajustar la velocidad relativa de cada motor suavemente según la magnitud del error.

Error = desviación de la línea


error = analogRead(SR) - analogRead(SL);
// SR > SL → robot fue a la derecha
// SL > SR → robot fue a la izquierda
// error ≈ 0 → robot centrado

Corrección diferencial


velIzq = velBase + corrección;
velDer = velBase - corrección;

analogWrite(ENA, velIzq); // PWM motor izq
analogWrite(ENB, velDer); // PWM motor der
Error positivo (SR ve más negro) → corrección positiva → velIzq sube, velDer baja → curva suave a la izquierda.

Los 3 Términos del Controlador PID

P
Proporcional

termP = kp × error;
Reacciona al error actual.
Más error → más corrección.
Kp alto: respuesta rápida pero oscilante.
Kp bajo: respuesta lenta y perezosa.
I
Integral

integrador += error;
termI = ki × integrador;
Acumula el error en el tiempo.
Corrige desviaciones persistentes.
Sin él el robot puede quedar torcido.
Ki alto: sobreoscilación.
D
Derivativo

termD = kd × (error - errorPrev);
errorPrev = error;
Reacciona a la velocidad de cambio.
Amortigua oscilaciones.
Frena la corrección antes de pasarse.
Kd alto: amplifica ruido.
Corrección total: corrección = kp×e + ki×Σe + kd×Δe

Etapa 05 — Código controlPID()


const float kp      = 0.15;  // ganancia proporcional
const float ki      = 0.002; // ganancia integral
const float kd      = 0.05;  // ganancia derivativa
const int   velBase = 180;   // PWM base (0-255)
const float limI    = 200.0; // límite anti-windup

float errorPrev  = 0;
float integrador = 0;

void controlPID() {
  float error = analogRead(pinSr) - analogRead(pinSl);

  integrador += error;
  integrador  = constrain(integrador, -limI, limI); // anti-windup

  float termP = kp * error;
  float termI = ki * integrador;
  float termD = kd * (error - errorPrev);
  errorPrev   = error;

  float corr = termP + termI + termD;
  int vIzq = constrain(velBase + (int)corr, 60, 255);
  int vDer = constrain(velBase - (int)corr, 60, 255);

  analogWrite(pinEna, vIzq);  // ENA → pin 9 PWM
  analogWrite(pinEnb, vDer);  // ENB → pin 10 PWM

  Serial.print("e:"); Serial.print(error,1);
  Serial.print(" P:"); Serial.print(termP,1);
  Serial.print(" I:"); Serial.print(termI,1);
  Serial.print(" D:"); Serial.print(termD,1);
  Serial.print(" vI:"); Serial.print(vIzq);
  Serial.print(" vD:"); Serial.println(vDer);
}

Etapa 05 — Cambio de Hardware Necesario ¡Importante!

En Etapas 1–4 los pines ENA y ENB del L293D van conectados a 5 V fijo → velocidad siempre máxima. Para PWM necesitamos pines con modulación.

Antes (Etapas 1–4)

  • ENA → 5 V directo
  • ENB → 5 V directo
  • Enable = siempre ON
  • Velocidad = fija (máxima)

Después (Etapa 5)

  • ENA → pin 9 (PWM) del Arduino
  • ENB → pin 10 (PWM) del Arduino
  • Enable = señal variable 0–255
  • Velocidad = controlada por PID

Calibración PID — Proceso Paso a Paso

① Solo P (Ki=0, Kd=0)

  • Aumentar Kp de 0.05 en 0.05
  • El robot debe empezar a seguir la línea
  • Parar cuando empiece a oscilar visiblemente
  • Anotar ese valor como Kp crítico

② Agregar D para suavizar

  • Aumentar Kd de 0.01 en 0.01
  • Las oscilaciones deben reducirse
  • Demasiado Kd → robot nervioso y ruidoso
  • Objetivo: respuesta rápida sin oscilar

③ Agregar I si hay desviación

  • Ki muy pequeño (0.001 – 0.005)
  • Corrige desviaciones en curvas largas
  • Si oscila de nuevo → reducir Ki

Serial Monitor — qué observar

e: 23.0 P: 3.45 I: 0.12 D: 0.80
vI: 204 vD: 156
  • e oscila ±20 → normal en curva
  • P domina → Kp bien ajustado
  • I crece sin freno → reducir Ki
  • vI/vD no deben saturar a 255/60
